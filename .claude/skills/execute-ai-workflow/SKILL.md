# Execute AI Workflow

Use this skill to run a previously created AI workflow. This skill enforces a strict separation between orchestration (main agent) and execution (sub-agents).

## Design Principles
- **No Templates/References**: This skill reads the routing rules directly from the specific workflow's `.ai-workflows/<workflow-id>/orchestration.md` and `steps/`.
- **Main Agent Orchestration:** The main agent owns routing logic, prepares inputs, generates a unique `run-id` (e.g., `run-20231027-143005`), and updates run state.
- **DAG Routing:** Step execution is resolved via a reactive DAG. A step is executed when its upstream `depends_on` steps have completed, and its `required_inputs` values match the expected conditions.
- **Sub-Agent Execution:** Every workflow step MUST be executed by a sub-agent. The sub-agent performs the `Instructions`, evaluates its `Success/Failure Criteria`, and returns a flat JSON output containing a `nextSteps` array.
- **Isolated Runs Directory:** Execution state and artifacts are stored in `runs/<workflow-id>/<run-id>/<step-id>/` at the **Project Root**.

## Execution Workflow

1. **Initialization / Resume:**
   - Resolve the workflow definition in `.ai-workflows/<workflow-id>/`.
   - **Generate `run-id`**: If starting a new run, generate a unique ID using the current date and time (e.g., `run-YYYYMMDD-HHMMSS`).
   - Ensure the execution directory exists: `runs/<workflow-id>/<run-id>/`.
   - Initialize `runs/<workflow-id>/<run-id>/state.md`. 
     - **Frontmatter**: Tracks JSON state, including `active_steps`, `completed_steps`, `pending_signals`, and `step_outputs`.
     - **Body**: Initialize with the workflow goal and a "Step Outcomes" header.
2. **Step Selection & Routing:**
   - Look at `active_steps` in `state.md`.
   - Provide the required data to the active step(s) by extracting from `step_outputs`.
3. **Delegation (Handoff) with Workspace Injection:**
   - Read the target step's `.md` file in `.ai-workflows/<workflow-id>/steps/`.
   - **Workspace Preparation**: Create the step-specific directory: `runs/<workflow-id>/<run-id>/<step-id>/`.
   - **Input Preparation**: Extract required `Input` values from previous outcomes in `state.md`.
   - **Mandatory `workDir`**: ALWAYS include `workDir` in the input context, set to the absolute path of the step-specific directory.
   - **Prompt Recording**: BEFORE spawning the sub-agent, you MUST construct the final, fully-resolved prompt and save it to `runs/<workflow-id>/<run-id>/<step-id>/sub-agent-prompt.md`. This is non-negotiable for auditability.
   - **Spawn Sub-Agent**: Use the final prompt to invoke a sub-agent. (Concurrent sub-agents can be spawned if multiple steps are active).
4. **Outcome Recording:**
   - Save the raw response to `runs/<workflow-id>/<run-id>/<step-id>/response.json`.
   - Verify the JSON structure against `.ai-workflows/<workflow-id>/steps/schemas/<step-id>.schema.json`.
   - **Update `state.md`**: 
     - Append the complete result to the body, including all returned fields and their descriptions. 
     - Call the `advance` command of the script to resolve DAG routing. It will append to `pending_signals`, verify `depends_on`, and validate `required_inputs`. If successful, the script updates `active_steps`. If dependencies are met but inputs are missing/mismatched, the workflow fails fast.
5. **Repeat:** Loop until `active_steps` is empty and `status` is completed or failed.

## Directory Structure
```text
(Project Root)
├── .ai-workflows/
│   └── <workflow-id>/
│       ├── orchestration.md
│       ├── steps/
│       └── workflow.spec.json
│
└── runs/                     <-- ALL execution state lives here
    └── <workflow-id>/
        └── <run-id>/         <-- Dynamically generated (e.g., run-20231027-143005)
            ├── state.md      <-- JSON Frontmatter for State Machine, Body for human-readable results
            ├── <step-01-id>/
            │   ├── sub-agent-prompt.md  <-- Mandatory: The exact prompt used
            │   ├── response.json        <-- Raw JSON response
            │   └── <artifacts...>       <-- Any files generated by the sub-agent
            └── <step-02-id>/...
```
